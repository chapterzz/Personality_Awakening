# T2.4 MBTI 评分算法服务设计说明

## 1. 目标

- 在 `apps/api` 提供可复用的 MBTI 计分能力，支持 `STANDARD` 与 `AVG` 两种来源。
- 对外提供统一 API：输入答案信号序列，输出四个维度分值与 `mbtiType`。
- 与 `PPA_Development_Task.md` 的 T2.4 验收对齐：输入答案序列，输出准确的人格类型和维度分值。

## 2. 输入/输出契约

### 2.1 输入（对外 API）

- 路径：`POST /scoring/mbti`
- 请求体：
  - `mode`: `'STANDARD' | 'AVG'`
  - `signals`: `Array<MbtiSignal>`
- `MbtiSignal` 结构：
  - `dimension`: `'EI' | 'SN' | 'TF' | 'JP'`
  - `side`: `'E' | 'I' | 'S' | 'N' | 'T' | 'F' | 'J' | 'P'`
  - `weight`: `1 | 2 | 3`

说明：

- 计分服务只消费“计分信号”，不直接依赖题库/CMS 的问题与选项结构。
- `STANDARD` 与 `AVG` 的原始 `answers` 到 `signals` 的映射放在上游（题库/CMS 或调用方）完成。

### 2.2 输出（对外 API）

- 成功响应：`{ success: true, data, message: 'ok' }`
- `data` 结构：
  - `mode`: `'STANDARD' | 'AVG'`
  - `scores`:
    - `EI`: `{ E: number, I: number, winner: 'E' | 'I', delta: number }`
    - `SN`: `{ S: number, N: number, winner: 'S' | 'N', delta: number }`
    - `TF`: `{ T: number, F: number, winner: 'T' | 'F', delta: number }`
    - `JP`: `{ J: number, P: number, winner: 'J' | 'P', delta: number }`
  - `mbti_type`: `string`（如 `INTJ`）

## 3. 计分规则

1. 初始化 8 个字母侧分值（`E/I/S/N/T/F/J/P`）为 0。
2. 遍历 `signals`，将每条信号的 `weight` 累加到对应 `side`。
3. 对每个维度分别决策 winner：
   - `EI`: 比较 `E` 与 `I`
   - `SN`: 比较 `S` 与 `N`
   - `TF`: 比较 `T` 与 `F`
   - `JP`: 比较 `J` 与 `P`
4. `delta = |left - right|`。
5. `mbti_type` 按维度顺序拼接：`EI` + `SN` + `TF` + `JP`。

## 4. 平局（tie-break）规则

- 当单维度两侧分值相等时，采用固定 deterministic 规则，取该维度左侧字母：
  - `EI` 平局取 `E`
  - `SN` 平局取 `S`
  - `TF` 平局取 `T`
  - `JP` 平局取 `J`

说明：

- 该规则保证同一输入必定得到稳定输出，便于测试与审计。
- 若后续 PRD/心理学专家给出强制偏置策略，可在不改外部契约的前提下替换本规则。

## 5. 错误语义

- DTO 校验失败（缺字段、非法枚举、空数组、weight 越界等）返回 `400`。
- 不可达分支（如 side 与 dimension 不匹配）在内部服务抛 `BadRequestException`，并返回统一错误体。

## 6. 模块边界

- `scoring.service.ts`: 核心算法（纯计算 + 最小输入一致性检查）。
- `scoring.controller.ts`: HTTP 层（DTO 校验、调用 service、响应封装）。
- `dto/compute-mbti.dto.ts`: 输入 DTO。
- `scoring.types.ts`: 领域类型（signal、scores、result）。

## 7. 测试策略

- 单元测试（Jest）：
  - 权重累加正确性
  - 四维 winner 计算
  - tie-break 规则
  - side 与 dimension 不匹配时抛错
- 集成测试（Supertest）：
  - `POST /scoring/mbti` happy path（`STANDARD`、`AVG` 各一例）
  - DTO 非法输入返回 400
