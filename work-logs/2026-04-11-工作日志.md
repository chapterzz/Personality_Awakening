## 今日工作（2026-04-11）

### 完成事项

- **Git 使用与环境**：说明安装后若终端找不到 `git`，多为 **PATH 未包含 Git** 或安装向导未选「命令行可用」类选项；给出用完整路径验证、重装改 PATH、或手动添加 `C:\Program Files\Git\cmd` 等处理方式。
- **版本管理范围**：明确执行 `git init` + 本地 `commit` 后，**版本历史仅在本机 `.git`**；未执行 `remote` + `push` 则 **GitHub 上无自动同步**。
- **`git add` 时 CRLF→LF 警告**：解释为 **行尾规范化提示**，一般无需单独处理；可选 `git add --renormalize .` 或编辑器统一 LF。
- **代码格式检查**：处理 `pnpm format:check` 对 **`docs/Git版本管理说明.md`** 的告警，通过 **`pnpm exec prettier --write`**（或 `pnpm format`）修复后 **`format:check` 通过**。
- **后端本地验收**：结合终端与浏览器验证 **`pnpm dev:api`** 行为：`http://localhost:3001` 根路径返回占位文案、**`/docs`** 为 Swagger UI、**`/docs-json`** 为 OpenAPI 3.0 JSON；说明误将 **`/docs` 与 `pnpm dev:web` 粘成 URL** 会导致 404，属操作/复制问题而非服务异常。
- **概念对齐**：解释本项目中 **Swagger / OpenAPI** 的含义及 **`/docs` 与 `/docs-json`** 的分工。
