### 遇到问题 & 解决方案

| 问题 | 处理 |
| :--- | :--- |
| 本机 **5432** 已被占用，`ppa-postgres` 无法绑定 | **`PPA_POSTGRES_PORT`**（及 Prisma **`DATABASE_URL`** 端口）与 Compose 一致；或释放 5432。 |
| Swagger **Authorize** 填了 JWT 但请求无 **`Authorization`** | OpenAPI operation 未挂 **`security`**；在 **`openapi.setup.ts`** 中为 `/progress` 设置 **`[{}, { 'access-token': [] }]`**（可选 Bearer）。 |
| Swagger 强制填 **`authorization` 参数** |去掉 **`@Headers('authorization')`**，改为 **`req.headers.authorization`**。 |
| **`Bearer JWT`** 被当成占位符导致 **`invalid_token`** | 使用真实 **`eyJ...`** token；**`node scripts/sign-dev-jwt.cjs <user-uuid>`**（`sub` 须为 **`users.id`**）。 |
| 根目录 **`require('jsonwebtoken')`** 报 **MODULE_NOT_FOUND** | pnpm 隔离；改用仓库 **`sign-dev-jwt.cjs`** 或 **`pnpm --filter api`** 下安装后再执行。 |
| 游客 **`PUT`** 误 **403** |修复 **`existing && existing.userId !== null`**（避免 `existing === null` 时 `undefined !== null` 误判）。 |
| 同一 **`session_id`** 再次 **`if_match_revision: 0`** 得 **409** | 预期乐观锁行为；新会话换新 **`session_id`** 或使用当前 **`progress_revision`**。 |
| **`pnpm add` 时 Prisma prepare EPERM**（Windows） | 关闭占用进程后重试；或使用无 **`jsonwebtoken`** 的签发脚本。 |
